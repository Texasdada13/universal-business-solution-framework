## The 7 Patterns Explained

### 1. Scraping (`scraping`)
**Copy from**: `patterns/scraping_patterns.py`
**Use for**: Getting data from websites
**Example**:
```python
from scraping_patterns import UniversalScraper

scraper = UniversalScraper(url="https://example.com/data")
data = scraper.scrape()
scraper.save_to_csv()
```

### 2. Change Detection (`change-detection`)
**Copy from**: `patterns/change_detection_patterns.py`
**Use for**: Tracking what's new or changed
**Example**:
```python
from change_detection_patterns import ChangeDetector

detector = ChangeDetector(data_dir='data')
changes = detector.detect_changes(data, key_fields=['id'])

if changes['new_count'] > 0:
    print(f"Found {changes['new_count']} new items!")
```

### 3. Email Alerts (`email`)
**Copy from**: `patterns/email_patterns.py`
**Use for**: Sending notifications
**Example**:
```python
from email_patterns import EmailAlerter

alerter = EmailAlerter()
alerter.send_new_items_alert(
    items=new_items,
    to_email="you@example.com",
    title="New Items Found"
)
```

### 4. Dashboard (`dashboard`)
**Generated by**: Project generator
**Use for**: Web interface with charts
**Includes**:
- Flask backend (`web/app.py`)
- HTML template (`web/templates/index.html`)
- CSS and JavaScript (`web/static/`)

### 5. Scheduler (`scheduler`)
**Copy from**: `patterns/scheduler_patterns.py`
**Use for**: Running tasks automatically
**Example**:
```python
from scheduler_patterns import UniversalScheduler

scheduler = UniversalScheduler(interval_minutes=60)
scheduler.set_task(my_function)
scheduler.start()
```

### 6. Analysis (`analysis`)
**Use for**: Processing data with pandas
**Includes**: Data transformation, validation, statistics

### 7. Documentation (`documentation`)
**Use for**: Generating reports
**Includes**: Markdown and Word document generation

## Example Workflows

### Workflow 1: Web Monitoring System


## Common Tasks

### Add a New Pattern
1. Create `patterns/my_pattern.py`
2. Add to `AVAILABLE_PATTERNS` in `create_project.py`
3. Document in `.claude/framework-context.md`
4. Test with example project

### Modify Existing Pattern
1. Edit file in `patterns/`
2. Test in an example project
3. Update documentation if behavior changed

### Share a Project
1. Ensure `.env` is in `.gitignore`
2. Include `requirements.txt`
3. Document setup in `README.md`
4. Commit to git

## Best Practices

### Start Small
- Use 1-2 patterns first
- Get it working
- Add more patterns later

### Test Each Component
- Run scraper alone first
- Test email separately
- Verify dashboard
- Then combine

### Use Environment Variables
- Never hardcode credentials
- Always use `.env` file
- Keep `.env.example` updated

### Document As You Go
- Update README with specific instructions
- Add comments to custom code
- Note any gotchas
